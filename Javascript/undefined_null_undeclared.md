# undefined & null & undeclared

## undefined

- 'undefined' 는 `변수가 선언되었으나, 아직 값이 할당이 되지 않았을때`의 기본값을 나타냅니다.
- 변수 선언 후 초기화 하지 않으면, 자동으로 'undefinde'값이 할당됩니다.

## null

- 'null'은 `변수가 값이 없음을 명시적으로` 나타내는 대 사용 됩니다.
- 개발자가 의도적으로 변수에 '비어있음'또는 '값 없음'의 상태를 표현할 때 'null'을 할당합니다.

## undeclared

- 'undeclared'상태는 `변수가 코드 내에서 선언되지 않았을 때`를 나타냅니다. 이 경우 변수는 어떠한 값도 할당받지 않았으며, 실제로 존재하지 않습니다.
- 'undeclared' 변수에 접근하려고 하면 'ReferenceError'가 발생합니다.
