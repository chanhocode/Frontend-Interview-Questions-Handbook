# let 과 var의 차이

1. Scope

   > let은 `Block Scope`를 가지고, var은 `Function Scope`를 가진다. </br>
   > 즉, let은 선언된 블럭 ( `{ }` ) 내에서만 유효하고 var은 선언된 함수 전체에서 유효하다.

2. 호이스팅 방식이 다르다.

   > `var`로 선언된 변수는 `선언과 초기화`가 함께 호이스팅 되어, 선언전 변수 사용시 `undefined`를 반환한다. `let`은 `선언`만 호이스팅 되어 선언전 변수 사용시 `참조에러`가 발생한다.

3. 재선언

   > `var`는 같은 스코프 내에서 `재선언` 할 수 있으며, `let은 할 수 없다.`

4. 전역객체와의 관계
   > 전역스코프에서 var로 선언된 변수는 전역객체(window, global)의 속성이 된다.

⚙️ 추가

- Scope는 무엇인가? <br>
  : 스코프는 식별자의 유효범위를 의미한다. 식별자는 자신이 어디에서 선언됐는지에 의해 다른 코드가 자신을 참조할 수 있는 범위를 갖을 수 있다.

- 변수의 선언, 초기화, 할당의 차이점 <br>
  : 변수의 선언은 변수를 생성하는 것을 의미하며 해당 식별자를 등록하여 스코프가 참조할 대상을 만듭니다. 초기화는 메모리에 변수 저장을 위한 공간을 확보하는 단계이며 기본적으로 undefined가 할당됩니다. 할당은 '=' 연산자를 사용하여 값을 할당하는 단계로, undefined로 초기화된 변수에 실제 값을 할당해주는 단계입니다.
