# 브라우저 저장소

## 로컬스트리지 (LocalStorage)

> `웹 브라우저에 데이터를 영구적으로 저장할 수 있다.` 사용자가 브라우저 캐시를 지우지 않는 한 데이터가 유지된다.

- 저장 용량이 5MB로 비교적 크다.
- 동일한 도메인 내의 모든 페이지에서 접근할 수 있다.

## 세션 스토리지 (SessionStroage)

> 세션 스토리지도 웹 브라우저에 데이터를 저장하지만, `사용자가 브라우저 창을 닫으면 저장된 데이터가 사라진다.`

- 각 브라우저 탭은 별도의 세션 스토리지를 가지며, 다른 탭 간에는 데이터를 공유하지 않습니다.
- 용량과 접근 방식은 로컬스토리지와 유사하다.

## Cookie

> `만료 기한이 있는 키값 저장소`이다. 쿠키는 서버와 클라이언트 간의 작은 데이터 조각을 저장하는 데 사용됩니다. (4KB)

- 쿠키는 클라이언트의 상태를 유지하기 위해 주로 사용되며, HTTP 요청과 함께 자동으로 서버로 전송된다.
- 쿠키는 주로 사용자 인증, 세션 관리등에 주로 사용된다.

⚙️ 추가

### 서버 인증과 브라우저 저장소

- 주요 정보를 요청 헤더에 넣는 방법
  > 보안에 매우 취약하다
- Session, Cookie 방식
  > 서버 부하가 증가하고, 세션 하이재킹 공격에 취약하다.
- JWT 방식
  > 별도의 브라우저 저장소에 저장하지 않고 JWT를 발급하고 검증하면 되어 확장성이 뛰어납니다. 그러나 Payload 정보가 제한적이고, JWT길이가 길다는 단점이 있다.
