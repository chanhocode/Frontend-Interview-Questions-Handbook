# SPA & CSR & SSR에 대해서

## SPA (Single Page Application)

> `최초 한번 페이지 전체를 로딩`한 후 부터 데이터만 변경해서 사용할 수 있는 웹 애플리케이션이다.

- SPA는 웹 애플리케이션 또는 웹사이트가 단일 페이지로 구성되어 있는 구조를 말한다.
- 대표적인 SPA 프레임워크 및 라이브러리로는 React, Angular, Vue.js 등이 있다.
- SPA는 `사용자 경험(UX)를 개선`하고 `빠른 페이지 전환`을 제공하지만, 초기 로딩 시 모든 자원을 로드해야 하며, `SEO(검색 엔진 최적화)에 약점`을 가질 수 있다.
- `단일 페이지에서 동적으로 콘텐츠를 갱신하는 방식으로, 사용자 경험에 중점을 둡니다.`

## CSR (Client-Side Rendering)

> 최초 로드시 필요한 파일들을 전부 받고, 사용자의 인터렉션에 따라서 클라이언트단에서 받아와 랜더링을 해주는 방식이다.

- CSR은 클라이언트 측에서 HTML을 렌더링하는 방식입니다.
- 서버에서는 초기 요청시 HTML 뼈대와 자바스크립트 파일을 전송하고, 클라이언트 측에서 자바스크립트가 실행되면서 페이지가 렌더링 됩니다.
- CSR은 SPA에서 주로 사용되는 렌더링 방식으로, 사용자 인터페이스의 상호작용이 매우 빠릅니다.
- 하지만 초기 로딩 속도가 느릴 수 있고, 검색 엔진 최적화에 불리할 수 있습니다.
- `클라이언트 측에서 HTML을 렌더링하여 빠른 사용자 인터페이스 상호작용을 제공하지만, 초기 로딩이 느릴 수 있습니다.`

## SSR (Server-Side Rendering)

> 요청시마다 새로고침이 일어나며 서버에 새로운 페이지에 대한 요청을 하는 방식이다.

- SSR은 서버 측에서 HTML을 렌더링하는 방식이다.
- 사용자의 요청에 따라 서버에서 각 페이지를 완전히 렌더링하고, 완성된 HTML 페이지를 클라이언트에 전송합니다.
- SSR은 초기 페이지 로딩 속도가 빠르고, SEO에 유리합니다.
- 하지만 페이지 전환 시마다 서버에 요청을 해야 하고, 서버에 부하가 갈 수 있으며, 페이지 전환 시 화면 깜빡임이 있을 수 있습니다.
- `서버 측에서 HTML을 렌더링하여 초기 로딩속도가 빠르고 SEO에 유리하지만, 페이지 전환시 마다 서버에 요청이 필요합니다.`

## MPA (Multiple Page Application)

> 사용자가 페이지를 요청할 때마다, 웹 서버가 요청한 UI와 필요한 데이터를 HTML로 파싱해서 보여주는 방식의 웹 어플리케이션 이다.

## SEO (Search Engine Optimization, 검색 엔진 최적화)

> SEO는 웹사이트나 웹페이지가 검색 엔진의 결과 페이지에서 더 높은 순위를 차지하도록 최적화 하는 과정이다. SEO에는 다양한 전략과 기술이 포함된다. SEO는 검색 엔진이 웹사이트를 쉽게 이해하고 색인화 할 수 있도록 돕는과정으로, 잘 실행된 SEO 전략은 웹사이트에 대한 트래픽을 증가시키고, 브랜드 인지도를 향상시킨다.
