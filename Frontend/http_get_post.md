# HTTP methods 에서 GET 과 POST 방식의 차이점

## GET 방식

- `동기식`
- 주로 서버로부터 정보를 조회하기 위해 사용
- 'GET'요청은 브라우저에 의해 `캐시될 수 있으며`, 브라우저 기록에 남고, 북마크할 수 있습니다.
- 'GET' 요청은 데이터의 길이에 제한이 있으며(브라우저 및 서버에 따라 다르다.), 보안상 민감한 데이터를 전송하기에 부적합하다.

## POST 방식

- `비동기식`
- 서버로 부터 데이터를 전송하여 새로운 리소스를 생성하거나 기존 리소스를 수정할 때 사용됩니다.
- `데이터는 HTTP메시지의 Body에 포함되어 전송`되므로 URL에 데이터가 노출되지 않습니다.
- 'POST'는 보다 안전하게 데이터를 처리할 수 있으며, 크기 제한이 없거나 훨씬 큰 데이터를 처리할 수 있습니다.
- 'POST' 요청은 브라우저 기록에 남지 않으며, `캐시되지 않습니다.`
