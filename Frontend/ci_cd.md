# CI와 CD에 대해서

## CI (Continuous Intergration, 지속적 통합)

> `지속적 통합은 개발 과정에서 코드 변경 사항을 주기적으로 메인 저장소에 통합하는 것`을 의미한다. 이 과정에서 코드 변경 사항이 자동으로 빌드되고 테스트 된다. CI의 주요 목표는 소프트웨어의 품질을 개선하고, 개발 과정에서 발생할 수 있는 문제를 조기에 발견하고 해결하는 것이다.

- `빌드와 테스트를 자동화해서 공유 저장소에 병합시키는 프로세스`

## CD (Continuous Delivery/Deploy, 지속적인 전달/배포)

> 지속적 배포는 모든 변경 사항이 자동으로 프로덕션 환경에 배포되는 과정을 말하고 지속적 전달은 소프트웨어가 언제든지 신뢰성 있게 프로덕션 환경에 배포될 수 있도록 준비하는 것을 의미한다.

- CD는 CI의 빌드/테스트를 통해서 정상적으로 수행됨을 확인하면 이는 배포를 수동으로 하느냐 자동으로 하느냐에 따라 2가지로 나뉨

- `지속적 전달`: 프로덕션 배포를 위한 상태가 되고 `배포 자체는 수동`으로 실행한다.
  - 개발팀과 비즈니스팀간의 커뮤니케이션 부족 문제를 해결한다.
- `지속적 배포`: `프로덕션까지 자동으로 배포`한다.
  - 어플리케이션의 제공 속도를 증가시킨다.
